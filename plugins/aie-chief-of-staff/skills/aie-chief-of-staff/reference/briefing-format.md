# Briefing format

Assemble the brief in this order. Keep it skimmable — most people read it in under two minutes. Adjust depth to `output.length` (brief / standard / detailed). Every news item and every claim carries a source link.

```
☕ DAILY BRIEF — {Weekday, Month D} · {name}

TOP 3 TODAY
  1. {the single most important thing} — {why / when}
  2. {second} — {why / when}
  3. {third} — {why / when}

🗓  SCHEDULE ({N} meetings · {first}–{last})
  {time}  {title}  ({internal/external})
          Attendees: {attendees}
          ⚑ Prep: {what to prepare}   ← only when prep is needed
  ...
  ⚠ {conflict / back-to-back / no-agenda flags}
  🎯 Focus block: {open window} — suggest protecting for {top priority}

📥 INBOX — {n} need action  (read-only; drafts on request)
  ★ {sender} — {one-line summary}
     → {suggested action: reply / delegate / schedule / archive}
  ...
  (Went quiet: {thread} — you owe {person} a reply since {day})

📰 INDUSTRY SIGNALS ({industry}, last {lookback})
  • {headline} — {why it matters to you}  [{source}]
  • {competitor move} — {so what}  [{source}]
  ...
  (Quiet day if little of substance — say so, don't pad.)

✅ SUGGESTED PLAN
  Morning:  {focus block → top priority}
  Before {first external meeting}: {prep item}
  Quick wins: {2-3 fast inbox actions}
```

## Rules
- **Lead with judgment, not a data dump.** The TOP 3 is the point; everything else supports it.
- **Prep flags only where real.** Don't annotate every meeting.
- **Tie news to the reader.** A headline with no "why it matters to you" gets cut.
- **Sources always.** Every industry item links out; inbox/calendar items reference the real thread/event.
- **Be honest about gaps.** If email or calendar isn't connected, say so and brief on what's available.
- **detailed** adds: full attendee lists, per-meeting context pulled from prior threads, and a longer news list. **brief** collapses to TOP 3 + schedule one-liners + top 3 news.
