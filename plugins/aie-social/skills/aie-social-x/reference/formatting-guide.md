# Formatting & Algorithm Guide (X, 2026)

## Character limits

- **Default: 280 characters per post.** Write to this so the output works for every account.
- Premium/verified accounts can post longer (up to 25,000 chars), but long single posts get truncated behind "Show more," which costs dwell. Default to 280-char units and thread instead. Only exceed 280 if the human says their account supports it and the content genuinely needs it.

## Single post vs thread — the decision

- **Single post** when the point is complete in one screen (under ~280 chars). Threads carry a drop-off cost: every extra post is another chance to lose the reader.
- **Thread** when you have 3+ distinct supporting points, a step-by-step, a story, or a list. Threads increase dwell time (a ranking signal) and generate more replies — but only if each post earns the next tap.
- **Thread length sweet spot: 5-9 posts.** For most topics 6-8 is ideal: a hook, 5-6 body posts (one idea each), and a closing CTA. Past ~10-12 posts you lose people unless the content is exceptional.

Threads with 6-9 posts generate meaningfully more saves than the same content as a single post.

## Thread structure

1. **Hook post** — stands alone, earns engagement by itself (see hook-library.md).
2. **Body posts** — one complete idea each: a tip, step, data point, or example. Readable in isolation.
3. **Closing post** — a clear CTA (follow for more / repost if it helped) plus any link.

Formatting inside a thread:
- Optional numbering (1/, 2/…) helps orientation on longer threads.
- Line breaks within a post for readability; don't wall-of-text a single post.
- Vary post length across the thread to keep rhythm.

## 2026 algorithm rules that change how you write

- **Replies are the currency.** The algorithm weights a back-and-forth reply far above a like (reports put a reply around +75 vs ~+0.5 for a like). Write to start conversations: take a stance, ask a real question, leave room to add on.
- **Saves signal value.** Teach something worth saving — steps, lists, frameworks — and the thread earns extended reach.
- **Dwell time.** "Did someone stop, read, and come back" is what the ranker rewards. That's why the hook, one-idea posts, and thread depth matter.
- **No external link in the main/hook post.** Links suppress reach. Put the link in the last post or a reply, and point readers to it.
- **Reply in the first hour.** Answering early replies lifts distribution. A posting behavior to flag for the human, not a copy change.
- **Engagement-farming is detected and suppressed** — "like if you agree, RT if you disagree" and similar. Use a genuine CTA instead.
- **Avoid ALL CAPS, dead-hour posting, and posting-and-ghosting** — all cited reach killers.

## Hashtags

- **0-1 maximum.** X uses semantic understanding to categorize content, so hashtags do little for discovery in 2026. 3+ can trigger spam filtering and reduce reach. Never put a hashtag in the hook.
